## Известные проблемы

### getPreviousCallSummaries возвращает транскрипты, не резюме
`supabase.js` → `getPreviousCallSummaries` выбирает поле `transcript`, а не `summary` из таблицы `calls`. Таблица `calls` вообще не имеет поля `summary`. Резюме хранится в `evaluations.raw_analysis`. Нужно либо добавить поле `summary` в `calls`, либо изменить запрос.

### Синхронизация двух репо фронтенда
Код в `admin/` и `denis-mutaf/leapy-admin` нужно синхронизировать вручную. Легко забыть.

### GLADIA_API_KEY в REQUIRED_ENV
Переменная `GLADIA_API_KEY` присутствует в `REQUIRED_ENV` в `index.js`, хотя Gladia больше не используется. Если на Railway убрать эту переменную — сервер упадёт. Нужно либо убрать из `REQUIRED_ENV`, либо удалить переменную из Railway и из кода.

## Архитектурные заметки для следующей сессии

При работе с analyze.js помни:
- `isFirstContact` = `client.total_calls === 0` (проверяется ДО `incrementClientCalls`)
- RAG context = semantic search по первым 500 символам транскрипта
- Claude возвращает JSON — всегда стрипаем markdown-фенсы перед `JSON.parse`
- `summary_metadata` содержит структурированные данные для автоматизации (теги, следующий шаг)

При работе с AmoCRM:
- `searchByPhone` пробует множество форматов номера (молдавский, российский)
- Заметка публикуется к сделке если есть `leadId`, иначе к контакту
- Тег "Квал" — только первый контакт, только если `status === 'relevant'` и `qualified !== false`
