Leapy — краткая память проекта
================================

Назначение
----------

Leapy — сервис, который **слушает звонки** отдела продаж компании **Isragrup** (девелопер жилья в Дурлештах, Кишинёв), **анализирует их через AI** и **обновляет AmoCRM** и собственную базу данных:

- принимает вебхуки от АТС **Moldcell Business PBX** (`POST /webhook`);
- скачивает MP3-запись звонка и транскрибирует её через **OpenAI gpt-4o-mini-transcribe**;
- анализирует разговор через **Anthropic Claude (claude-haiku-4-5-20251001)**:
  - пишет резюме звонка в стиле РОПа;
  - оценивает работу менеджера по 5 критериям (1–10);
  - извлекает ключевые инсайты о клиенте;
- сохраняет всё в **Supabase** (PostgreSQL + pgvector + Storage);
- публикует заметку, теги и задачи в **AmoCRM**;
- отдельно предоставляет **RAG-API** для работы с базой знаний (документы, поиск, Q&A), админка к которой живёт в **Next.js**.

Основная бизнес-цель: дать руководителю и менеджерам **быстрый обзор качества звонков и статуса клиентов** без ручного ввода заметок, и одновременно иметь **обучаемую базу знаний** по продукту.

Технологический стек
--------------------

- Backend: **Node.js 20**, **Express 4**, ES Modules (`"type": "module"`), деплой на **Railway**.
- AI:
  - OpenAI:
    - `gpt-4o-mini-transcribe` — транскрипция аудио;
    - `text-embedding-3-small` — эмбеддинги (1536 dim) для RAG;
  - Anthropic:
    - `claude-haiku-4-5-20251001` — резюме и оценка звонков.
- БД: **Supabase** (PostgreSQL + pgvector, проект `oznbosvtlrdwbnmonoie`).
- Хранилище: Supabase Storage (`call-recordings`, `rag-documents`).
- CRM: **AmoCRM** (`deniskosharny.amocrm.ru`).
- Frontend admin: **Next.js 15** (App Router) + Tailwind, деплой на **Vercel** (`leapy.vercel.app`), репо `denis-mutaf/leapy-admin` (зеркало папки `admin/` в основном репо).

Высокоуровневая архитектура
---------------------------

- `index.js` — точка входа Express:
  - проверяет обязательные env (`PBX_CRM_TOKEN`, `OPENAI_API_KEY`, `ANTHROPIC_API_KEY`, `GLADIA_API_KEY`, `AMO_LONG_TOKEN`, `AMO_SUBDOMAIN`, `SUPABASE_URL`, `SUPABASE_SERVICE_KEY`);
  - CORS-middleware с `ALLOWED_ORIGINS`;
  - маршруты:
    - `GET /health` — healthcheck;
    - `POST /webhook` → `src/webhook.js`;
    - `/rag/*` → `src/rag-routes.js`.
- `src/webhook.js`:
  - принимает вебхук от PBX, сразу отвечает 200;
  - скачивает MP3 во временный файл `/tmp`;
  - загружает запись в Supabase Storage (`call-recordings/<year>/<callid>.mp3`);
  - создаёт или находит клиента по телефону, находит менеджера по `pbx_user`;
  - создаёт запись звонка в таблице `calls` (status: `processing`);
  - транскрибирует аудио (OpenAI `gpt-4o-mini-transcribe` с доменным промптом под IsraGrup);
  - достаёт контекст предыдущих звонков клиента и передаёт всё в `analyzeTranscript`;
  - сохраняет:
    - оценку менеджера в `evaluations`;
    - структурированные инсайты в `call_insights`;
    - обновляет профиль клиента в `clients` и счётчик `total_calls`;
  - интеграция с AmoCRM:
    - находит/создаёт контакт и сделку;
    - публикует заметку с резюме и оценкой;
    - обновляет имя контакта/сделки, если узнали имя клиента;
    - при **первом контакте** ставит тег **«Квал»**, если лид квалифицирован;
    - создаёт follow-up задачу, если в анализе есть следующий шаг.
- `src/analyze.js`:
  - выполняет **два параллельных вызова Claude**:
    - `summarizeCall` — резюме в стиле РОПа (первичный / повторный звонок);
    - `evaluateManager` — оценка по 5 критериям (1–10) + рекомендации;
  - подмешивает **RAG-контекст** из `rag.js` (semantic search по первым 500 символам транскрипта);
  - различает **первый контакт** и **повторный** по `client.total_calls === 0`:
    - первый — полный шаблон (клиент, запрос, финансы, возражения, следующий шаг);
    - повторный — только «дельта» (что нового, новые возражения, изменение температуры, новый шаг);
  - возвращает единый объект `analysis`, который используется в webhook:
    - `summary` (строка для заметки в AmoCRM),
    - `evaluation` (баллы и рекомендации),
    - `client_insights` (интерес, бюджет, возражения, заметки),
    - `has_next_step`, `next_step_text`, `next_step_deadline_days`,
    - `summary_metadata` (температура, qualified/неквал, и т.д.).
- `src/rag.js` + `src/rag-routes.js`:
  - парсинг документов (PDF/DOCX/TXT/HTML/MD);
  - чанкинг: 1500 символов, overlap 200;
  - embeddings через `text-embedding-3-small`;
  - сохранение в Supabase: `documents` + `document_chunks (vector(1536))`;
  - эндпоинты: загрузка, список, удаление, семантический поиск, генерация названия, RAG Q&A.
- `src/supabase.js`:
  - все обращения к БД: клиенты, менеджеры, звонки, оценки, инсайты, загрузка аудио;
  - RPC `increment_client_calls` или fallback read-then-write.
- `src/amocrm.js`:
  - HTTP-клиент к AmoCRM с токеном `AMO_LONG_TOKEN`;
  - поиск контакта по телефону (разные форматы Молдова/Россия);
  - создание контакта+сделки, заметки, задач, тэгов, обновление имён.

RAG и админ-панель
------------------

- RAG-часть живёт в бэкенде (`/rag/*`) и используется как:
  - API для админки (`admin/`, Next.js);
  - источник контекста для анализа звонков (`analyze.js`).
- Админ-панель (`leapy-admin`):
  - секции:
    - загрузка документа (drag-and-drop, генерация названия через Claude);
    - список документов;
    - «Спросить AI» (RAG Q&A, ответ + источники);
    - тестовый поиск (сырые чанки с similarity);
  - обращается к бэкенду через `NEXT_PUBLIC_API_URL` (по умолчанию `https://leapy-production.up.railway.app`).

Секреты и окружения
-------------------

Критичные переменные окружения (Railway / Supabase / Vercel):

- `PBX_CRM_TOKEN` — shared secret с Moldcell PBX для валидации вебхуков.
- `OPENAI_API_KEY` — для транскрипции и эмбеддингов.
- `ANTHROPIC_API_KEY` — для вызова Claude.
- `AMO_LONG_TOKEN`, `AMO_SUBDOMAIN` — доступ к AmoCRM API.
- `SUPABASE_URL`, `SUPABASE_SERVICE_KEY` — полный доступ к БД и Storage (service_role).
- `ALLOWED_ORIGINS` — whitelist CORS для фронтенда (пусто → все).
- `NEXT_PUBLIC_API_URL` (на фронте) — URL бэкенда.

Важно: service_role в Supabase даёт полный доступ, поэтому **бэкенд — единственная точка входа**; прямой доступ к Supabase из браузера не используется.

Инварианты и ожидания от AI
---------------------------

1. Язык:
   - Клиенты и менеджеры могут говорить на **русском**, **румынском** или смешанно.
   - Резюме звонка должно быть **на преобладающем языке разговора** (если 50/50 — на русском).
   - Все служебные комментарии/рекомендации для команды — на **русском**.
2. Поведение при анализе звонков:
   - Нельзя придумывать факты: бюджет, сроки, имя клиента и т.п. берутся только из транскрипта.
   - Если информация не прозвучала — поле либо не выводится, либо явно отмечается как «не озвучен».
   - При повторных звонках резюме должно содержать **только изменения** относительно прошлого.
3. Работа с RAG:
   - Ответы в `/rag/ask` должны опираться **только на найденные чанки**.
   - Если релевантных чанков нет — возвращать пустой `sources[]` и аккуратный ответ без галлюцинаций.
4. Безопасность:
   - Номер телефона — ключевая привязка клиента во всех системах (Supabase, AmoCRM).
   - Любые изменения в схеме БД или структуре анализа должны быть задокументированы в `docs/` (особенно в `BACKEND.md`, `DECISIONS.md`, `CONTEXT.md`).

Кратко: как объяснить Leapy «в одном предложении»
-------------------------------------------------

> Leapy — это связка **PBX → AI → CRM + RAG**, которая автоматически превращает телефонные звонки в структурированные заметки, оценки менеджеров и обновлённые профили клиентов в AmoCRM, опираясь на Supabase как основное хранилище данных и OpenAI/Claude как мозг системы.
